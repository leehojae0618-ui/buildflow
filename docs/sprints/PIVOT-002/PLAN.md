# PIVOT-002 Plan

## Objective

Create the first working Requirement layer that converts a user Goal into a structured Requirement Snapshot, identifies Clarification Questions, assesses Constraints, and lists Consent Requirements.

## Product Principle

질문의 품질이 설계의 품질을 결정한다. BuildFlow should understand before it recommends or builds.

## Technical Scope

- Goal Parser
- Requirement model
- Clarification model and question generation
- Constraint assessment model
- Consent requirement model
- Snapshot creation
- Project creation/update Server Action integration
- Project detail Requirement summary
- Unit tests

## Out of Scope

- Architecture
- Build Plan
- Prompt generation
- Build Engine
- Marketplace
- New Migration or database table
